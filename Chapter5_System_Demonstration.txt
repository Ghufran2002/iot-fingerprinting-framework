5.4 SHAP EXPLAINABILITY DEMONSTRATION
=======================================

5.4.1 Live Explainability Panel
---------------------------------
Figure 5.7 presents the SHAP (SHapley Additive exPlanations) Live Explainability
Panel from the Plotly Dash dashboard. This panel addresses the critical
requirement of transparency in AI-based security systems — it explains *why*
the model made a specific prediction, not merely *what* it predicted.

The panel header reads:
  "Live Explainability — Why the Model Made Its Last Prediction"
  "Each bar shows a feature's SHAP contribution to the current device prediction.
   Green = pushes model TOWARD that device type. Red = pushes AWAY."

The SHAP analysis shown corresponds to a flow classified as "Unknown" with 46%
confidence. The top-10 contributing features ranked by absolute SHAP value are:

  Feature               | Direction  | Interpretation
  ----------------------|------------|------------------------------------------
  tcp_ratio             | Red (Away) | High TCP usage pushes away from prediction
  udp_ratio             | Red (Away) | High UDP ratio atypical for this device
  ack_ratio             | Red (Away) | ACK patterns inconsistent with device
  is_https              | Red (Away) | HTTPS presence reduces confidence
  mean_dest_port        | Red (Away) | Destination port profile does not match
  well_known_port_ratio | Red (Away) | Well-known port usage too high/low
  syn_ratio             | Green      | SYN ratio supports this device class
  is_mqtt               | Green      | MQTT traffic partially consistent
  is_encrypted          | Red (Away) | Encryption profile deviates from baseline
  max_pkt_size          | Green      | Maximum packet size aligns with device

The dominance of red (away) contributions explains the low confidence score
(46%) and the "Unknown" classification — the flow's protocol and port
characteristics do not sufficiently match any of the eight trained device
profiles, triggering the low-confidence fallback mechanism.

5.6 DISCUSSION
===============

The live demonstration validates the end-to-end functionality of the proposed
framework. Several key observations are noted:

1. Stability: The system served 106 API requests over 7,551 seconds of
   continuous operation without any crashes or performance degradation,
   demonstrating production-level stability.

2. Accuracy: The 100% anomaly detection rate during the demonstration session,
   combined with the 100% fingerprinting accuracy established during training
   and testing, confirms the efficacy of the ensemble approach.

3. Explainability: The SHAP panel successfully identified the most influential
   features for each prediction in real time, fulfilling the transparency
   requirement that distinguishes this framework from conventional black-box
   intrusion detection systems.

4. Operational Readiness: The combined deployment of FastAPI and Plotly Dash
   through a single launch command (python run.py) confirms that the system
   is ready for operational deployment in a smart home or laboratory environment.

5. Threshold Effectiveness: The 0.75 anomaly threshold correctly separated
   anomalous traffic (scoring 0.75–0.97) from normal traffic (scoring below
   0.75), as evidenced by the gauge reading of 0.361 for a normal flow and
   alert scores consistently above 0.80 for injected attacks.
